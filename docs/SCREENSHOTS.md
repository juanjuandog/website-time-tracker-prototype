# Screenshot Checklist

Use this checklist before making the repository public or sharing it widely.

## Required

1. `docs/screenshots/dashboard.png`
   - Show the main dashboard with today / week / month controls.
   - Prefer clean demo data.
   - Avoid private browser tabs, names, emails, or sensitive websites.

2. `docs/screenshots/category-prompt.png`
   - Show the floating desktop category prompt.
   - Use a harmless sample site or app name.

3. `docs/screenshots/markdown-report.png`
   - Show a rendered Markdown report or preview.
   - Include daily / weekly / monthly summaries if possible.

## Nice To Have

4. `docs/screenshots/menu-bar.png`
   - Show the companion menu with open dashboard, pause, autostart, and quit.

5. `docs/screenshots/demo.gif`
   - Short flow: visit unknown website, classify it, see dashboard update.

## Tips

- Use demo data rather than real personal browsing data.
- Keep the browser window clean and not too zoomed out.
- Crop screenshots so the app is the focus.
- Prefer PNG for static screenshots.
- Keep file names lowercase with hyphens.
